Templates

PawPantry uses Shopify's Online Store 2.0 JSON template format. Each template maps a URL route to an ordered set of sections.

Templates live in templates/. JSON templates (.json) define which sections are rendered and in what order. Liquid templates (.liquid) are used for pages that need tightly coupled logic.

ℹ️
Alternate templates

You can create alternate templates for any type directly in Shopify admin (e.g. product.no-sidebar.json) and assign them to individual products or collections.

Store templates

File Route Description
index.json / Homepage. Contains hero, benefits bar, featured products, categories grid, and testimonials. All sections can be reordered or hidden in the editor.
product.json /products/* Individual product page. Renders main-product with gallery, variant picker, and reviews.
collection.json /collections/* Collection listing page. Renders main-collection with filters, sorting, and a product grid.
cart.json /cart Shopping cart page. Renders main-cart with line items, delivery progress bar, and checkout.
search.json /search Search results. Renders main-search showing matched products, pages, and articles.
page.liquid /pages/* Generic page template. Renders the title and rich-text body content set in the Shopify admin.
page.contact.json /pages/contact Contact page. Renders main-contact. Assign this template to your contact page in the Shopify admin.
404.liquid 404 routes Not-found page with a friendly message and a link back to the shop.
gift_card.liquid /gift_cards/* Gift card redemption page displaying the card code and balance.
password.liquid /password Password-protected store page shown when the store is not yet open to the public.

Customer account templates

All customer account pages live in templates/customers/. They share the same header and footer as the rest of the store.

File Route Description
login.liquid /account/login Customer login form with email and password. Includes "Forgot password?" and a registration prompt.
register.liquid /account/register New customer registration with first name, last name, email, and password.
account.liquid /account Account dashboard showing recent orders and links to addresses and order history.
order.liquid /account/orders/* Individual order detail with line items, shipping address, payment method, and order status.
addresses.liquid /account/addresses Address book where customers can add, edit, or remove saved delivery addresses.
activate_account.liquid /account/activate Account activation page shown when a customer is invited by the store owner.
reset_password.liquid /account/reset Password reset form shown after a customer follows the "Forgot password?" email link.

Layout

layout/theme.liquid is the master wrapper for every page. It outputs the <html>, <head>, and <body> tags, loads assets/theme.css and assets/theme.js, and renders the header, announcement bar, main content block, and footer.

⚠️
Edit theme.liquid with care

Changes to layout/theme.liquid affect every page. An unclosed tag or malformed Liquid expression can break the entire storefront.

Localisation

All user-facing strings are defined in locales/en.default.json — navigation labels, cart copy, product page text, form labels, and accessibility strings. To customise a string without editing source code, edit the locale file directly. To add a new language, duplicate the file and translate the values (keys must remain unchanged).

Key group Covers
generalPassword page meta text
navigationMenu labels: Home, Shop, Contact, Account, Wishlist, Cart
cartCart empty state, delivery progress, totals, checkout button
productAdd to basket, Sold out, star ratings, tabs (Ingredients, Nutrition, Reviews), wishlist
